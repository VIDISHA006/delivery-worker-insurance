# Delivery Worker Insurance (Parametric Model)

## 🚨 Problem
Delivery workers face income loss due to accidents, bad weather, or unexpected events.

## 👤 Target Users
- Food delivery riders
- Courier partners

## 💡 Solution
A weekly micro-insurance platform that provides instant payouts based on predefined triggers.

## 🔄 Workflow
1. User signs up
2. Pays weekly premium (₹20–₹50)
3. System monitors triggers
4. If trigger occurs → automatic payout

## 💰 Weekly Premium Model
- ₹30 per week
- Affordable for gig workers
- Payout: ₹500–₹2000 depending on event

## ⚡ Parametric Triggers
- Rainfall > 50mm → payout
- Government curfew → payout
- App downtime > 2 hours → payout
- Accident detection → payout

## 🤖 AI/ML Usage
- Fraud detection
- Risk scoring of users
- Dynamic premium adjustment

## 📱 Platform Choice
Mobile app (because delivery workers primarily use phones)

## 🛠 Tech Stack
- Frontend: Flutter
- Backend: Node.js
- Database: Firebase
- APIs: Weather API, Maps API

## 📈 Future Scope
- Integration with Swiggy/Zomato
- Real-time tracking
